# Copywriting Rules

> Visuals get the click. Copy gets the swipe, the save, the comment, the DM. The hook is the first sentence; the rest is whether you can pay it off.

This file defines:

1. **Hook physics** — the rules of the first slide.
2. **Slide-level copy rules** — what goes on each slide.
3. **Caption rules** — how to write the longer-form caption that lives below the post.
4. **CTA rules** — how to convert attention into action.
5. **Voice and rhythm** — how to sound human, not like an AI marketer.

---

## Hook physics

A hook must do three things, in this order:

1. **Stop the scroll.** Pattern interrupt. Specificity. Tension. Threat or promise.
2. **Promise a payoff.** The reader should know roughly *what they'll get* by swiping.
3. **Make scrolling feel costly.** "If I don't read this, I'll regret it" is the goal.

If the hook does only #1, it's clickbait and the reader bounces on slide 2.
If the hook does only #2, it's a brochure and the reader doesn't stop.
If the hook does only #3, it's a threat and the reader resents you.

You need all three. Compressed into ≤14 words.

### The hook formula

```
[SPECIFIC SUBJECT] + [TENSION] + [IMPLIED PAYOFF]
```

Examples (using placeholders):

- "Most [AUDIENCE] still believe [COMMON_BELIEF]. That's why they're stuck at [PROBLEM]." → contrarian + audience identification + payoff implied
- "I lost [TIMEFRAME] to [MISTAKE]. Don't be me." → confession + warning + payoff implied
- "There's a feature in [PRODUCT] that 90% never open." → specificity + scarcity + payoff implied
- "[N] [THINGS] tested. [M] survived." → data + drama + payoff implied

### Hook patterns that work

| Pattern | Example template |
|---|---|
| **Status quo callout** | "Most [AUDIENCE] still [DEFAULT_BEHAVIOR]. Here's what they're missing." |
| **Confession** | "I [DID_THING] for [TIMEFRAME] before I figured out [INSIGHT]." |
| **Insider knowledge** | "[CATEGORY] insiders quietly do [THING]. Beginners don't." |
| **Threshold moment** | "If you're still [BEHAVIOR] in [CURRENT_YEAR], stop now." |
| **Counter-intuitive number** | "[METRIC] that gets ignored. It's worth [CONSEQUENCE]." |
| **Sharp opinion** | "[POPULAR_THING] is overrated. Here's what to do instead." |
| **Visible cost** | "[BEHAVIOR] is costing [AUDIENCE] [METRIC] every week." |
| **Public test** | "I tested [N] [THINGS] for [TIMEFRAME]. The winner surprised me." |
| **Frame upgrade** | "Stop thinking of [X] as [OLD_FRAME]. It's [NEW_FRAME]." |
| **Specific outcome** | "How [PERSON/COMPANY] [DID_THING] in [TIMEFRAME] without [PAIN_POINT]." |

### Hook patterns that fail

| Pattern | Why it fails |
|---|---|
| "Are you tired of...?" | Old infomercial cadence. Triggers skip reflex. |
| "Did you know that...?" | Reads as trivia. No tension. |
| "5 tips that will change your life" | Generic. No specificity. No tension. |
| "Here's a thread on..." | Inflates expectation, delivers a list. |
| "🔥 INSANE 🔥" / all-caps screaming | Reads as low-effort spam. |
| "Hot take:" / "Unpopular opinion:" | Telegraphs that what follows isn't actually hot or unpopular. |
| "Buckle up." / "Strap in." | Filler. Adds words, not value. |
| "Most people don't realize..." | Vague. Replace "most people" with "most [AUDIENCE]." |

### Specificity scale

Run your hook through this scale. Aim for level 4 or 5.

| Level | Example |
|---|---|
| 1 — Vague | "Get more done in less time." |
| 2 — Slightly specific | "Save time on meetings." |
| 3 — Audience-specific | "Most engineers waste 3 hours a week on standup." |
| 4 — Audience + mechanism | "Most engineers waste 3 hours a week on standup. The fix takes 5 minutes." |
| 5 — Audience + mechanism + tension | "Most engineers waste 3 hours a week on standup. The fix takes 5 minutes — and your manager will hate it." |

The jump from level 3 → 4 is mechanism. Level 4 → 5 is tension. Both are paid in specificity.

## Slide-level copy rules

### One idea per slide

If a slide has two sentences carrying two unrelated ideas, split it. If a slide has one idea but says it three times, cut to one.

A test: can you write the slide's idea in 4 words? If you can, the slide is focused. If you need 12, the slide is mushy.

### Headline + support, never headline + paragraph

The slide is the album cover. The caption is the liner notes. Don't put a paragraph on the slide.

```
Headline: 6-12 words
Support text: optional, max 8 words
```

Anything longer goes in the caption.

### Use plain language

If your audience is highly technical, you can use technical vocabulary. But avoid:

- Marketing jargon ("synergy," "leverage," "unlock," "supercharge," "transform")
- LinkedIn-bro cadence ("Here's the thing.")
- Empty intensifiers ("absolutely," "literally," "100%")
- Fake mystery ("the one secret nobody wants you to know")
- Filler transitions ("So, here's what I learned...")

### Show numbers, not adjectives

"Massive improvement" → dead. "47% faster" → alive. "Drops average response time from 2.4s to 1.1s" → alive and credible.

If you don't have the number, write `[PROOF_NEEDED: short description]` and surface it in the Missing Inputs section.

### Read the slide out loud

If you stumble reading it, rewrite it. Social copy is *spoken-feeling* even when written.

### Use the rule of three sparingly

Lists of three are persuasive. They're also overused. Default to a single idea, beautifully said, before reaching for a tricolon.

## Voice and rhythm

### Defaults

- Conversational. You're talking to one person, not a conference room.
- Short sentences. Mix in a longer one occasionally so the rhythm doesn't feel staccato.
- Lowercase-friendly when the brand allows it.
- Active voice. "We rebuilt the onboarding" beats "The onboarding was rebuilt."
- First-person singular ("I") for founder posts. First-person plural ("we") for company posts. Choose and stay consistent in a single carousel.

### Override defaults if the Brand Pack says so

The Brand Pack's `voice.md` is law. If it specifies sentence-case, formal register, or third-person voice, follow it.

### Cadence patterns

- **Punch-pause-punch:** "We shipped it. Three customers cancelled. Best decision we made."
- **Statement-question:** "Most teams default to X. What if X is the bottleneck?"
- **Setup-twist:** "The dashboard was beautiful. Nobody opened it."

Avoid:

- LinkedIn poetry voice (each. line. on. its. own.) unless the Brand Pack explicitly asks for it.
- Long, dependent-clause-laden sentences that bury the verb.
- "Hopefully," "kind of," "maybe" — hedging that signals lack of conviction.

## Caption rules

The caption is what holds the conversation. It does three jobs:

1. **Reinforce the carousel's argument** in your own voice (≠ repeating the slides).
2. **Give the reader a reason to comment.**
3. **Drop a single, clear CTA.**

### Length

- LinkedIn: 150-250 words is the sweet spot for thought-leadership.
- Instagram: 100-180 words. Hook the first 2 lines hard — the rest is hidden behind "more."
- X (carousel): 150-260 characters per slide if multi-tweet; for image carousels, 100-140 words in the post.
- TikTok: 80-140 characters; the on-slide hook does most of the work.

### Structure

```
[Line 1 — restate the hook in a different way]
[Line 2-3 — the strongest single sentence from the carousel]

[Body — 80-150 words: the argument, one human story or example, one short reframing line]

[Line N-2 — the conversation prompt: a question or invitation]
[Line N-1 — the CTA]
[Line N — optional: signature line / brand line / hashtag line]
```

### Conversation prompts that work

- "What's the one [THING] you'd kill in your stack today?"
- "Tell me a [THING] I missed in the comments."
- "Drop your [TOOL/PROCESS/OPINION] below — I want to compare notes."
- "If you've shipped [THING], what would you tell someone starting?"

### Conversation prompts that fail

- "Thoughts?" (too low-effort)
- "Agree or disagree?" (false binary)
- "Let me know in the comments!" (no specific question)
- "Hope this helps!" (signs off without inviting reply)

## CTA rules

The CTA is the post's job. It must be one of these — never two:

- **Save** ("Save this for your next [DECISION_POINT].")
- **Share** ("Send this to [AUDIENCE_DESCRIPTION] who's making this mistake.")
- **Comment** ("Drop your [THING] in the comments.")
- **Click** ("Full guide / dataset / repo at the link in bio / first comment.")
- **Follow** ("Follow for more [TOPIC] without the [PAIN_POINT].")
- **DM** ("DM me ['KEYWORD'] and I'll send you [THING].")
- **Try** ("Try [PRODUCT] free at [URL].")

### CTA matching

| Carousel angle | Best CTAs |
|---|---|
| Contrarian | Comment, Share |
| Mistake | Comment, Save |
| Hidden Feature | Save, Try |
| Before/After | Save, DM |
| Comparison | Save, Comment |
| Myth vs Reality | Share, Save |
| Benchmark | Click (full data), DM |
| Build in Public | Comment, Follow |
| Founder | Comment, Follow |
| Technical Breakdown | Save, Click |
| Workflow | Save, DM |
| Checklist | Save, Share |
| Opinionated Ranking | Comment, Share |
| You're Using X Wrong | Save, Comment |
| Stop Doing X | Comment, Save |
| Nobody Tells You | Save, Share |
| I Tested X | Click, DM |

### Bad CTAs to kill

- "Hope this was helpful!" (no action)
- "Let me know your thoughts." (no specific action)
- "Like and follow for more!" (low signal)
- "What did I miss?" (only works if asked once per quarter)

## First-comment rules

The first comment is real estate. Use it for one of:

1. **The link** the platform demoted in the post.
2. **A bonus** ("If you want the full template, here it is.")
3. **A contrarian footnote** ("One thing I left out of the carousel: [NUANCE].")
4. **A receipt** (a screenshot, a quote, a data point that backs up the claim).

Length: 30-80 words.

Bad first comments: copy-paste of the caption, "great post 🙌," or the entire carousel transcribed as text.

## Alternative hooks (always generate 5)

Every output includes 5 alternative hooks. Use them to:

- A/B test on different days.
- Repurpose the same content with a new lens.
- Adapt for different platforms.

Make the alternatives genuinely different — not minor wording changes. Switch the angle, the audience, or the framing for each.

Example set for a carousel about "the meeting that should have been a doc":

1. **Confession:** "We ran a 90-minute meeting last week. The first half could have been a paragraph."
2. **Contrarian:** "Stop calling them 'sync meetings.' They're paid focus groups for indecision."
3. **Mechanism:** "The 30-second test that decides if your meeting should be a doc."
4. **Identity:** "How senior operators kill meetings before they're scheduled."
5. **Number-led:** "The 4 questions that turn 80% of your standing meetings into 5-minute docs."

## Anti-patterns to avoid

A short list of patterns that mark a post as AI-generated within 3 seconds:

- Starting every slide with the same word ("Stop. Stop. Stop.")
- Ending every slide with an em dash and a clever phrase.
- Excessive em dashes generally — yes, even this one.
- Three-sentence paragraphs in the caption that all start with "And."
- "Game-changer," "supercharge," "transform," "unleash," "seamless," "robust."
- "In today's fast-paced world,..."
- "Imagine a world where..."
- Closing the caption with a string of unrelated hashtags.
- Calling the audience "friends" or "fam."
- Emoji in every line.
- "Crazy how nobody talks about this 👇"

If your draft contains any of these, rewrite. Do not ship.

## The 60-second copy edit

Before you finalize a carousel, run this pass:

1. Read slide 1 alone. Would you stop?
2. Read slides 2-3. Are you hooked or bored?
3. Read slide 4. Does it answer slide 3?
4. Skim every headline. Could a tired person follow the argument from headlines alone?
5. Read the caption out loud. Did you stumble?
6. Look at the CTA. Is it one action, not two?
7. Check the words-to-avoid list from the Brand Pack. Any violations?
8. Search for marketing jargon. Replace with plain language.

If steps 1-3 fail, restart the carousel. Don't try to save a weak hook with strong slides 4-8.
