# Carousel Structures — Module Cheat Sheet

> Quick reference. The full structures live in [`system/carousel-framework.md`](../../system/carousel-framework.md).

---

## Default (8 slides) — the workhorse

```
1. Hook
2. Problem
3. Mistake
4. Reframe
5. Mechanism
6. Example / Proof
7. Application
8. CTA
```

When in doubt, use this.

---

## Launch (10 slides)

```
1. Hook
2. Why now
3. The old way
4. The cost of the old way
5. Introducing [PRODUCT/FEATURE]
6. How it works (mechanism)
7. What it replaces
8. Use case 1
9. Use case 2
10. CTA
```

For product / feature / brand launches.

---

## Educational (9 slides)

```
1. Hook
2. Why this matters
3. The concept named
4. Component 1
5. Component 2
6. Component 3
7. How to put it together
8. Where it breaks (be honest)
9. CTA
```

For frameworks, methods, concepts.

---

## Comparison (8 slides)

```
1. Hook
2. Dimensions that matter
3. Option A overview
4. Option B overview
5. Where A wins
6. Where B wins
7. Recommendation by use case
8. CTA
```

For X vs Y posts. Be even-handed.

---

## Benchmark / Proof (9 slides)

```
1. Hook (what tested, sample size)
2. Methodology
3. Headline result
4. Breakdown segment 1
5. Breakdown segment 2
6. The surprise
7. Implication
8. Limitations
9. CTA
```

For data-driven posts. Methodology earns the rest.

---

## Thought Leadership (8 slides)

```
1. Hook (strong opinion)
2. Status quo position
3. Steelman the status quo
4. Where it breaks
5. New position in detail
6. Pattern from your experience
7. The principle, generalized
8. CTA
```

For founder / executive takes.

---

## Product-Led (10 slides)

```
1. Hook (workflow / outcome)
2. Job to be done
3. Default tools / workflow
4. Why default fails
5. Enter [PRODUCT]
6. Feature 1
7. Feature 2
8. Feature 3
9. Concrete result
10. CTA
```

When the product is the protagonist.

---

## Project Announcement (8 slides)

```
1. Hook (what it is)
2. Why we built it
3. What's in v1
4. How it works (one diagram)
5. Who it's for
6. Who it's not for
7. What's next
8. CTA
```

For OSS / side-project / public artifact releases.

---

## Picking a structure

If the user specifies → honor it.
If `auto` → match the goal:

| Goal | Structure |
|---|---|
| save | default / educational |
| comment | thought-leadership / contrarian (use default with contrarian angle) |
| share | comparison / benchmark |
| click | benchmark / launch |
| dm | thought-leadership / launch |
| follow | thought-leadership / build-in-public (use default) |
| try | launch / product-led |
| awareness | thought-leadership / project-announcement |

---

## Counts

- Default: 8.
- Launches & product-led: 10.
- Educational & benchmark: 9.
- Lighter (problem-aware audience already): 7.
- 11+: never. Split into a series.
