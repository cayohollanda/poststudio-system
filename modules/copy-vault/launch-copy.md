# Launch Copy — Pattern Library

> Patterns for product / feature / brand / project launches. Honest, specific, no fake urgency.

---

## Hook patterns

### Why-now-led
```
[CATEGORY] changed when [SHIFT]. We built [PRODUCT/FEATURE] for what comes next.
```

```
The week we shipped [FEATURE], [USE_CASE] became 10x cheaper. Here's why.
```

### Outcome-led
```
[CONCRETE_WORKFLOW] now takes [SHORT_TIMEFRAME]. Here's what changed.
```

```
What we built instead of [DEFAULT_TOOL].
```

### Confessional-led
```
We almost shipped [WRONG_VERSION]. Here's why we didn't, and what we shipped instead.
```

---

## Body patterns

### Old way → cost → new way → mechanism → use cases
```
Slide N: "The old way: [DEFAULT_WORKFLOW]."
Slide N+1: "What it cost: [METRIC]."
Slide N+2: "The new way: [PRODUCT]."
Slide N+3: "How it works: [MECHANISM]."
Slide N+4: "Use case 1 — [SCENARIO]."
Slide N+5: "Use case 2 — [SCENARIO]."
```

### Honest scope
```
Here's what [PRODUCT] does:
- [CAPABILITY]
- [CAPABILITY]
- [CAPABILITY]

Here's what it doesn't:
- [NON-CAPABILITY]
- [NON-CAPABILITY]

This is for [AUDIENCE] when [SITUATION]. If you're [DIFFERENT_AUDIENCE], use [ALTERNATIVE].
```

This honest-scope pattern earns trust faster than a feature dump.

## CTA patterns

```
Try [PRODUCT] free at [URL]. Link in bio.
```
```
[PRODUCT] is free for [USE_CASE]. Open at [URL].
```
```
Star the repo at [URL] if it solves [PROBLEM] for you.
```
```
Join the waitlist at [URL]. We open access in [TIMEFRAME].
```

## Banned launch tropes

- "🚀 The future of [CATEGORY] is here!"
- "We're so excited to announce..."
- "After months of hard work..."
- "Game-changing new [PRODUCT]"
- "Revolutionary"
- "Unleash"
- "Disrupting [CATEGORY]"
- Countdowns timed to fake urgency
- "Limited spots — only [N] left" (unless literally, demonstrably true)
- "The Tesla of [CATEGORY]" / similar comparison flexes

---

## Required honesty in a launch

Every launch post should answer:

1. **What is it, in one line?** (no marketing fog)
2. **Who is it for?** (specific segment)
3. **Who is it not for?** (specific segment)
4. **What does it cost?** (or "free" if true)
5. **What's the mechanism?** (why does it work)
6. **What's the trade-off?** (be honest)

Without these, the launch reads as a press release. With them, it reads as a real product.
