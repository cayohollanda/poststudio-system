# Objections — Pattern Library

> Patterns for handling audience objections. Use in caption, in CTA-adjacent slides, or in dedicated objection-handling campaigns.

---

## The 4 most common objections

For any product / service / idea, audiences resist with one of:

1. **"It's not for me."**
2. **"It costs too much / takes too long."**
3. **"I've tried something like it."**
4. **"I don't need it right now."**

For each, we have a counter-pattern.

---

## "It's not for me"

### Acknowledge-then-redirect
```
You're right that [APPROACH] isn't for everyone. Specifically, it's not for [NON_AUDIENCE]. It is for [PRIMARY_AUDIENCE] — and the difference is [SPECIFIC_TRAIT].
```

### Specificity-as-permission
```
If you're [ROLE] at [STAGE/SIZE], this is for you. If you're [DIFFERENT_ROLE], skip this and read [ALTERNATIVE].
```

### Honesty-as-trust
```
Most [AUDIENCE_BROAD] don't need [APPROACH]. The ones who do are [SPECIFIC_SEGMENT]. If that's not you, this is the wrong post.
```

---

## "It costs too much / takes too long"

### Compared-to-doing-nothing
```
Doing nothing isn't free. The compounding cost of [DEFAULT_BEHAVIOR] is [METRIC] over [TIMEFRAME]. Compare that.
```

### Asymmetric-investment
```
[INTERVENTION] takes [SHORT_TIMEFRAME]. The cost it removes runs every [TIMEFRAME] indefinitely.
```

### Structural-vs-behavioral
```
Behavioral fixes ("just be more disciplined") cost daily. Structural fixes cost once.
```

---

## "I've tried something like it"

### Honest-difference
```
[ALTERNATIVE] solves [DIFFERENT_PROBLEM]. We solve [OUR_PROBLEM]. The category looks similar; the job-to-be-done isn't.
```

### Trade-off-named
```
[ALTERNATIVE] wins on [DIMENSION]. We win on [DIFFERENT_DIMENSION]. Pick by which dimension actually blocks you.
```

### Mechanism-mismatch
```
[ALTERNATIVE]'s mechanism is [MECHANISM_A]. Ours is [MECHANISM_B]. They produce similar surfaces and very different outcomes.
```

---

## "I don't need it right now"

### Threshold-named
```
You don't need [APPROACH] yet. You'll need it when [SPECIFIC_TRIGGER]. Save this for then.
```

### Wait-or-prevent
```
Wait until [PROBLEM_HAPPENS], or prevent [PROBLEM] now. Either is fine; the cost is the same.
```

---

## Anti-patterns

- **Bullying the objection.** "If you don't see the value, you're the problem." → never.
- **Over-explaining.** Long objection-handling reads as defensive.
- **Dismissive replies.** "We've heard that before." → patronizing.
- **Inventing a fictional persona who objected.** "A customer told me X." → only if real and verbatim.

---

## When to dedicate a post / slide / campaign to objection handling

- The same objection appears in 3+ comments / DMs.
- A campaign's penultimate post is the natural place for the top objection.
- Post-launch week is the natural moment for objection-handling content.
