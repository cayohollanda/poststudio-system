# Contrarian Copy — Pattern Library

> Patterns for opinionated, status-quo-challenging content. Confidence with mechanism. No edginess for its own sake.

---

## Hook patterns

### Status-quo callout
```
Most [AUDIENCE] still believe [COMMON_BELIEF]. That's why they're stuck at [PROBLEM].
```

### Sharp opinion
```
[POPULAR_THING] is overrated. Here's what to do instead.
```

### Threshold
```
If you're still [BEHAVIOR] in [CURRENT_YEAR], stop.
```

### Frame upgrade
```
Stop treating [X] as [OLD_FRAME]. It's [NEW_FRAME].
```

### Inverted advice
```
Everyone says [POPULAR_ADVICE]. They're wrong because [MECHANISM].
```

---

## Body patterns

### Steelman → counter
Always steelman the opposing position before countering it. Otherwise you're shadowboxing.

```
Slide N (steelman): "The case for [POPULAR_BELIEF] is real: [VALID_REASON]."
Slide N+1 (counter): "But it breaks at [SPECIFIC_CONDITION], because [MECHANISM]."
```

### Mechanism-led contrarian
```
The default looks right because [SURFACE_EXPLANATION]. The actual driver is [DEEPER_MECHANISM]. So the default optimizes the wrong thing.
```

### Personal authority + opinion
```
I've shipped [N] [PROJECT_TYPE]s. Here's the unpopular take: [OPINION].
```

This requires real personal authority. Without it, the opinion reads as hot-take noise.

## CTA patterns

```
Comment your strongest counter. I'll engage.
```

```
Save this for the next [DECISION_POINT] when you're tempted to do [DEFAULT_BEHAVIOR].
```

```
Share with the [AUDIENCE] who needs to read it.
```

## Anti-patterns

- **Contrarian without mechanism.** "Everyone is wrong about X." → vague. Why are they wrong?
- **Contrarian as posture.** Disagreement for its own sake reads as performance.
- **Contrarian against straw men.** "People say X" — does anyone actually say X? Verify.
- **Contrarian against named individuals.** Direct call-outs by name read as drama. Address the position, not the person.
- **Contrarian with no proof.** Sharp opinion without evidence ages badly.
- **"Hot take:" / "Unpopular opinion:" prefixes.** They telegraph that the take isn't actually hot or unpopular.

---

## Calibration rule

A contrarian post should make ~10-20% of the audience uncomfortable. If it makes 0%, it's not actually contrarian. If it makes 50%+, it's likely cruel or wrong.

The discomfort is *productive*: it's the audience having to update a belief.

---

## What makes a contrarian post safe

- The brand has personal experience with the position.
- The mechanism is named and defensible.
- The opposing position is steelmanned before being countered.
- Specific (not "everyone is wrong" but "most [AUDIENCE] in [SITUATION] are wrong").
- No personal attacks.
- No fabricated metrics in support of the opinion.

---

## When NOT to be contrarian

- The brand has no authority on the topic.
- The position can't be defended with mechanism.
- The audience is already in agreement (preaching to the choir = no tension).
- The contrarian take contradicts the Brand Pack's `constraints.md`.
- The brand is in legal / regulated / sensitive category where opinions carry risk.

A weak contrarian post is worse than a non-contrarian one. It teaches the audience the brand is loud without being right.
