# Input Collection Checklist (per cycle)

> What we need from the client at the start of each content cycle (typically monthly). Without these, generation produces weaker output.

---

## Required (per month)

- [ ] **Cycle theme / focus** — one sentence.
- [ ] **Upcoming events / launches / proof drops** — what's happening this month.
- [ ] **New proof captured since last cycle** — fresh quotes, metrics, customer stories.
- [ ] **What worked / didn't last cycle** — feedback on shipped content.
- [ ] **Audience reactions captured** — comments, DMs, reactions worth saving.
- [ ] **Off-limits topics for the period** — anything sensitive this month.
- [ ] **Capacity confirmation** — same cadence as last month, or different?

---

## Optional

- [ ] Specific topics the client wants covered.
- [ ] Specific founder appearances scheduled (talks, interviews) — the brand should signal-boost.
- [ ] New competitors / category developments to consider.
- [ ] New customer interviews available for proof updates.

---

## How we collect

A monthly 30-min check-in call OR an async form sent at the end of each month. Pick one based on client preference.

---

## What we do with the inputs

1. Update `brands/[slug]/proof-assets.md` with new proof.
2. Update `brands/[slug]/examples.md` with new working / non-working post records.
3. Update `brands/[slug]/voice.md` if new audience-adopted vocabulary surfaced.
4. Update `brands/[slug]/rejected-examples.md` with patterns that didn't work.
5. Refresh the next month's plan via `prompts/12-create-monthly-content-plan.md`.

---

## When inputs are missing

- 1-2 missing items → ship with `[MISSING_INPUT]` flags in the next plan.
- 3+ missing items → recommend a working session before content goes live.
- Same items missing for 2 cycles in a row → escalate; the engagement isn't getting the input it needs.

---

## Cycle cadence

- **Monthly cycle:** input collection at month-end → next-month plan ready by day 1 → content shipping by day 3.
- **Bi-weekly cycle:** light input every 2 weeks; full plan refresh monthly.
- **Quarterly cycle:** light input monthly; full strategy refresh quarterly.
